# i18n — 국제화 규칙 (I18N_LOCALE)

> Loomix v10.0 다국어 지원 기반. 현재 지원: `ko` (기본) · `en`.

## 환경 변수

```bash
I18N_LOCALE=ko   # 한국어 (기본값, 미설정 시 ko 사용)
I18N_LOCALE=en   # 영어
```

## 파일 위치

```
.claude/i18n/ko.json   — 한국어 메시지 번들
.claude/i18n/en.json   — 영어 메시지 번들
```

## 로딩 규칙

1. `I18N_LOCALE` 환경변수 확인
2. 미설정 또는 `ko` → `.claude/i18n/ko.json` 로드
3. `en` → `.claude/i18n/en.json` 로드
4. 키 없음(fallback) → 한국어 번들 참조

## 메시지 포맷

`{변수명}` 플레이스홀더 사용:

```python
# 예시
msg = i18n["pipeline"]["step_failed"].format(step=8, reason="FFmpeg timeout")
# → "Step8 실패: FFmpeg timeout"
```

## 에이전트 사용 지침

- 사용자에게 출력되는 **알림·요약·리포트** 메시지는 번들에서 참조
- 코드 주석·로그·SSOT JSON은 언어 무관 (한국어 고정, CLAUDE.md 규칙 준수)
- 새 메시지 추가 시 ko.json + en.json 동시 업데이트 필수

## 신규 언어 추가

1. `.claude/i18n/{locale}.json` 생성 (ko.json 구조 복사)
2. `I18N_LOCALE={locale}` 환경변수 정의
3. 이 파일의 지원 언어 목록 업데이트
4. devops-engineer에게 PR 리뷰 요청
