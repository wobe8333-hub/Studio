테스트 검증 시, 가장 중요한 사용 흐름은?

파이프라인을 실행할 때 어떤 방식으로 검증하고 싶으신가요?

A

실시간 관찰 중심

파이프라인 실행 → 각 Step 진행상황을 실시간으로 보면서 → 스크립트·이미지·영상이 생성되는 과정을 단계별로 확인

  • Step별 진행바 / 현재 작업 표시
  • 생성 즉시 이미지·스크립트 미리보기
  • 실시간 로그 스트리밍
B

결과물 리뷰 중심

파이프라인 실행 → 완료 후 → 모든 결과물(스크립트, 이미지, 영상, 썸네일)을 한 페이지에서 종합 검수

  • 채널별 / Run별 결과물 카드
  • 이미지·썸네일 갤러리 비교
  • QA 점수 & 승인/거부 버튼
C

단계별 수동 제어

각 Step을 개별적으로 트리거 → 결과 확인 → 다음 Step 진행 여부 결정 (단계 건너뛰기 / 재실행 가능)

  • Step00~17 개별 실행 버튼
  • 각 Step 입력/출력 JSON 뷰어
  • 실패 Step 재시도 UI
